## 1. Contexto y problema

Las 17 MCP tools de devbrowser (`browser_*`) exponen un `inputSchema` **inválido** porque el
helper `EncodeSchema` (en `mcp_utils.go`) **serializa el struct de args con valores cero**
(`{"fullpage":false}`, `{"url":""}`, …) en vez de un JSON Schema. Clientes MCP como Claude Code
descartan TODO el `tools/list` si una tool es inválida → el agente no ve ninguna tool.

**La generación del JSON Schema NO es responsabilidad de devbrowser.** Ahora `tinywasm/mcp` la
genera a partir del modelo de args (`Tool.Args model.Fielder` → `Schema()`). devbrowser solo debe:
(a) tener modelos de args bien escritos al estándar nuevo, y (b) declarar `Args` en cada tool.

Además, `models.go` está escrito con el **estilo antiguo** (tags de formulario `input:"-"` /
`input:"number"` / `help:`), que es un concern de UI irrelevante para argumentos MCP.

## 2. Cambios

### 2.1 `models.go` — autorizar los modelos como `model.Definition` (patrón OFICIAL)

**Sigue la documentación oficial de `tinywasm/model`** (léela antes de escribir):
- `model/README.md` → sección *"Defining a Model Definition"*.
- `model/docs/API_FIELD.md` → `Definition`, `Fields`, `Field`, `FieldType`.

Según la doc: *"The source of truth for a model is now a `Definition` literal. From a `Definition`,
the `ormc` generator produces the concrete Go struct and its zero-reflection plumbing. This inverts
the previous flow (struct + string tags → generated schema)."*

Por tanto `models.go` debe contener **solo literales `model.Definition`** (la fuente de verdad),
NO structs con tags. El struct concreto (`ScreenshotArgs`, …), su `Schema()`, `Pointers()`,
`Validate()` y el codec los **genera ormc** en `models_orm.go`.

**Convención de nombres** (idéntica a `mcp/model.go`, que es la referencia): la variable
`<StructName>Model` genera el struct `<StructName>`; `Definition.Name` es la identidad en snake_case.

Reescribe `models.go` así (una `Definition` por cada tipo de args actual; conserva los mismos
nombres de campo en el wire — snake_case, p.ej. `wait_after`, `await_promise`):

```go
package devbrowser

import "github.com/tinywasm/model"

var ScreenshotArgsModel = model.Definition{
	Name: "screenshot_args",
	Fields: model.Fields{
		{Name: "fullpage", Type: model.FieldBool},
	},
}

var ClickElementArgsModel = model.Definition{
	Name: "click_element_args",
	Fields: model.Fields{
		{Name: "selector", Type: model.FieldText, NotNull: true}, // NotNull → required en el JSON Schema y Validate() generado
		{Name: "wait_after", Type: model.FieldInt},
		{Name: "timeout", Type: model.FieldInt},
	},
}

var NavigateArgsModel = model.Definition{
	Name: "navigate_args",
	Fields: model.Fields{
		{Name: "url", Type: model.FieldText, NotNull: true},
	},
}
// … una Definition por cada args actual: EmulateDeviceArgs, GetConsoleArgs, FillElementArgs,
//    SwipeElementArgs, EvaluateJSArgs, GetNetworkLogsArgs, GetErrorsArgs, GetPerformanceArgs,
//    GetContentArgs, InspectElementArgs, GetSourceArgs, GetStylesArgs, GetStorageArgs,
//    GetAssetArgs, InterceptRequestArgs. Mapea el tipo Go → FieldType:
//    string→FieldText, int→FieldInt, bool→FieldBool. Marca NotNull los campos obligatorios.
```

Referencia de `FieldType` (de `model/docs/API_FIELD.md`): `FieldText`, `FieldInt`, `FieldFloat`,
`FieldBool`, `FieldBlob`, `FieldStruct`, `FieldIntSlice`, `FieldStructSlice`, `FieldRaw`.

> Los tags de formulario `input:"..."`/`help:"..."` del `models.go` viejo desaparecen: son UI, no
> argumentos MCP. Si en el futuro se necesita un widget, se pone en `Field.Widget` (no en tags).

### 2.2 Regenerar `models_orm.go` con ormc

Añade el marcador de generación y ejecuta ormc (patrón oficial del ecosistema, p.ej.
`tinywasm/user/generate.go`):

```go
// generate.go
package devbrowser

//go:generate ormc
```

Ejecuta `go generate ./...` (o el binario `ormc` de `github.com/tinywasm/orm/ormc`). ormc escanea
las `model.Definition` de `models.go` y **genera** en `models_orm.go` los structs (`ScreenshotArgs`,
…) con `Schema() []model.Field`, `Pointers()`, `EncodeFields`/`DecodeFields` y `Validate()`.
NO edites `models_orm.go` a mano (cabecera `DO NOT EDIT. generated by ...`). Los call sites
`new(ScreenshotArgs)` siguen compilando porque el struct ahora vive en el archivo generado.

### 2.3 Borrar `mcp_utils.go` (`EncodeSchema`)

Elimina el archivo `mcp_utils.go` completo. devbrowser ya no genera JSON Schema.

### 2.4 Declarar `Args` en cada tool (17 archivos `mcp-*.go`)

Reemplaza en cada `mcp.Tool{...}`:

```go
// ANTES
InputSchema: EncodeSchema(new(ScreenshotArgs)),
// DESPUÉS
Args: new(ScreenshotArgs),
```

Para tools sin argumentos reales (las que usaban `EmptyArgs`/`Reserved`), pon `Args: nil` (o
simplemente omite el campo). mcp emitirá `{"type":"object","properties":{}}` automáticamente.

### 2.5 Bump de `mcp`

Sube `github.com/tinywasm/mcp` en `go.mod` a la versión publicada con `Tool.Args` (ver
MASTER_PLAN). `go mod tidy`.
